1. MANUAL PARA ADMINISTRADOR

El administrador tiene acceso completo al sistema, incluyendo gestión de usuarios, tickets, dependencias, reportes, backups y logs de auditoría.

1.1 Inicio de Sesión

1.1.1 Acceder al Sistema

  1. Abra su navegador web (Chrome, Firefox, Edge, Safari)
  2. Diríjase a la dirección del sistema CSI
  3. Ingrese su usuario y contraseña
  4. Haga clic en "Iniciar Sesión"

1.2 Gestión de Tickets

1.2.1 Crear un Ticket

  1. Haga clic en "Nuevo Ticket" en el menú lateral
  2. Complete los campos obligatorios:
    • Dependencia donde se presenta la falla: Seleccione la ubicación física del problema
    • Lugar/Área: Especifique el lugar exacto (número de oficina, sala, etc.)
    • Área de Soporte: Seleccione el tipo de soporte necesario
    • Servicio Específico: Seleccione el servicio
    • Asunto: Título breve del problema
    • Descripción: Detalle completo del problema
  3. Si desea, adjunte archivos (imágenes, documentos)
  4. Configure la prioridad (Baja, Media, Alta, Urgente)
  5. Opcionalmente, active "Asignar técnico inmediatamente" para asignar desde la creación
  6. Haga clic en "Crear Ticket"

4. REFERENCIAS GENERALES

4.1 Estados de Tickets

Estado Descripción Color
Nuevo El ticket acaba de ser creado y no tiene técnico asignado Azul
Asignado Un técnico ha sido asignado al ticket Naranja
En Proceso El técnico está trabajando en resolver el problema Verde
Cerrado Exitosamente El problema fue resuelto ✅ Gris oscuro
Cerrado No Exitoso No fue posible resolver el problema Rojo oscuro

Flujo de un Ticket

NUEVO → ASIGNADO → EN PROCESO → CERRADO EXITOSAMENTE / NO EXITOSO │ │ │ │ │ │ │ └── ✅ Problema resuelto / ❌ No resuelto │ │ └── Técnico trabajando en la solución │ └── Técnico asignado al ticket └── Ticket recibido, sin asignar

4.2 Prioridades

Prioridad Descripción Tiempo Sugerido
Urgente Problema crítico, atención inmediata 1-2 horas
Alta Problema importante, atender urgente 4-8 horas
Media Problema moderado, atender pronto 1-2 días
Baja Problema menor, puede esperar 3-5 días

4.3 Privilegios de Usuario

Privilegio Descripción Permisos
Administrador Control total del sistema Gestionar usuarios, tickets, dependencias, backups, ver logs
Técnico Atención de tickets Ver/asignar tickets, cambiar estados, cerrar tickets
Usuario Reportar problemas Crear tickets, ver tickets propios, seguimiento

4.4 Consejos Útiles

Para todos los usuarios:

  • ✅ Use un asunto descriptivo para identificar rápidamente su ticket
  • ✅ Proporcione detalles completos en la descripción
  • ✅ Si es posible, adjunte capturas de pantalla del error
  • ✅ Sea específico en la ubicación del problema

Al reportar problemas técnicos:

  • Error en equipo: Indique el número de inventario si lo conoce
  • Error en sistema: Indique qué acción estaba realizando
  • Error en red: Indique desde dónde intenta conectarse

Ubicación del problema:

Sea lo más específico posible: "Oficina 201, área de atención al público" es mejor que simplemente "Oficina".
